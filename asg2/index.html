Conversations with Robots: Voice, Smart Agents & the Case for Structured Content

  1. What is the difference between structured content and semantic HTML?

    Primarily, Structured Content is more so the information itself- which is framed like as articles, recipes, product descriptions, profiles, and how-to’s, which deal with the hierarchy of the content for the viewers to see. Semantic HTML, on the other hand, hierarchizes the information within the code itself, such as code lists or heading tags like "h1” & “li” , which allows for both the people and the algorithms can recognize them as elements which are both defined and interpretable.
    Additionally, Structured Content is generally explicitly defined and described between the various chunks of content data so as a result, which allows for the algorithm to be able to legibly interpret the code in a way that allows it to find and shoot out answers by being able to target specific related areas to search between the database. While, Semantic generally acts to communicate information between just the document elements themselves, rather than just describing how they should appear on screen.

  2. How do they work together?

    Overall, they can work together to combine their particular areas of coverage to create a hierarchical format for both the code and information, that is efficient for both the algorithm and people to view at. For, the semantic will be mostly working to categorize the information of the code itself, whilst the structured content will act similarly to a blueprint of how the information should be organized in order to give out targeted information to the viewers. With these two working together, the information itself and within the code will be having the shared characteristics of semantic’s defined interpretability and the structured content’s explicitly defined legibility.

  3. Why is it important to have structured content and semantic HTML in your web pages?

    It’s crucial to have both semantic HTML and Structured Content in the web pages mainly for the very same reasons mentioned in the previous question above because they both are quite helpful in creating an end result that has both of the qualities: such as the Structured Content’s ability to give out targeted answers which are tailored by having an explicit categorized order of the information and the Semantic HTML’s allowance for interpretability in mostly the behind the scenes code which sets the groundwork for both people and algorithms alike to go through and recognize the elements. Also, like the intro of the article also mentioned, “by focusing on providing access to information in a structured, systematic way that is legible to both humans and machines, content publishers can ensure that their content is both accessible and accurate in these new contexts”, which enforces that it’s important to have a blend of both so the information can be both hierarchized and categorized for the viewer’s benefit, whether it is the algorithm or coder going through, or the viewer looking for answers.

  2. In David McRaney's article, what is "hyperbolic discounting"? List one method McRaney identifies to counteract hyperbolic discounting. Which would work best for you?

    As defined in David McRaney’s article, Hyperbolic Discounting is a term to describe the effect it has upon one’s mindset, which incites one to act in favor of the short term ideal, rather than abiding by the true better payout which happens more long term. In other terms, it creates a free void for the mind to put off things, so that the person doesn’t feel the pressure of the long term unless it becomes immediate, which allows for one to dismiss all of the aspects which they don’t wish to deal with in the immediate moment to later times.
    One of the methods which was mentioned in the article, which would work the best for me, was the idea of the set-up of zones, which could significantly invoke a space to force one’s self to perform the task, instead of having it being pushed away inside of the void of hyperbolic discounting. With this, it’ll make the task seem more immediate, thus making this particular task be the better payout in the short term, as well as the long term. Additionally, setting up zones that are set for before the actual date would also work as a way to portray self control because the zones are set up knowingly to be deliberately before the actual final date, however, the work will still be completed before then from a deliberate attempt of creating this as the most crucial immediate, above all others.
